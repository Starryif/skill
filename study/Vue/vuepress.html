    # 基础环境

    git (opens new window):版本管理

    yarn (opens new window):npm包管理的替代

    git --version 查看Git版本

    # 搭建

    1. 创建Github (opens new window)帐号
    2. 新建一个项目（仓库）
      • 勾选with a README
      • 勾选add .gitignore Node
    3. 通过SSH地址克隆（下载）项目到本地
      本地新建一个文件夹，右键git bash here打开命令行，输入：
    git clone git@github.com:Starryif/skill.git
    1. 初始化文件夹
    yarn init # npm init

    目录下生成文件package.json

    1. 将VuePress安装为本地依赖（官方建议做法）
    yarn add -D vuepress # npm install -D vuepress

    (选择全局安装的话)

    yarn global add vuepress # npm install -g vuepress
    1. 创建第一篇文档
    mkdir docs && echo '# Hello VuePress' > docs/README.md
    1. 在 package.json 中添加一些scripts
    #package.json
{
	"scripts": {
		"docs:dev": "vuepress dev docs",
		"docs:build": "vuepress build docs"
	}
}
    1. 在本地启动服务器
    yarn docs:dev # npm run docs:dev

    VuePress 会在 http://localhost:8080 启动一个热重载的开发服务器

    # 导航栏

    1. 目录结构的约定
      vuepress遵循“约定优于配置”的原则
    2. 页面路由的映射规则
    文件的相对路径 页面路由地址
    /README.md /
    /guide/README.md /guide/
    /config.md /config.html
    1. 导航栏的配置（全局、文件）

    静态资源目录：.vuepress/public
    增加导航栏 Logo

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    logo: '/assets/img/logo.png',
  }
}

    配置导航栏链接
    <a>标签的特性将默认包含target="_blank" rel="noopener noreferrer",
    可另外提供targer与rel，将被作为特性增加到<a>标签。

    // .vuepress>config.js
module.exports= {
   themeConfig: { 
        nav:[
             {text:"home",link:"/"},
             {text:"guide",link:"/guide/"},
             {text:"external",link:"https://baidu.com",target:"_self",rel=""},
       ]
   }
}

    当提供一个items数组而不是一个单一的link时，将显示为一个下拉列表。

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    nav: [
      {
        text: 'Languages',
        ariaLabel: 'Language Menu',
        items: [
          { text: 'Chinese', link: '/language/chinese/' },
          { text: 'Japanese', link: '/language/japanese/' }
        ]
      }
    ]
  }
}

    可以通过嵌套的items来在下拉列表中设置分组

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    nav: [
      {
        text: 'Languages',
        items: [
          { text: 'Group1', items: [/*  */] },
          { text: 'Group2', items: [/*  */] }
        ]
      }
    ]
  }
}

    全局禁用导航栏

    // .vuepress/config.js
module.exports = {
  themeConfig: {
    navbar: false
  }
}

    禁用某个指定页面的导航栏

    ---
navbar: false
---

    # 侧边栏

    1. 侧边栏的作用：
      • 文章内快速检索（目录TOC）
      • 关联内容快速跳转（导航）
    2. 侧边栏分组
      通过使用对象来将侧边栏划分成多个组
    // .vuepress/config.js
module.exports = {
  themeConfig: {
    sidebar: [
      {
        title: 'Group 1',   // 必要的
        path: '/foo/',      // 可选的, 标题的跳转链接，应为绝对路径且必须存在
        collapsable: false, // 可选的, 默认值是 true,
        sidebarDepth: 1,    // 可选的, 默认值是 1
        children: [
          '/'
        ]
      },
      {
        title: 'Group 2',
        children: [ /* ... */ ],
        initialOpenGroupIndex: -1 // 可选的, 默认值是 0
      }
    ]
  }
}
    1. 多个侧边栏
      为不同的页面显示不同的侧边栏
    // .vuepress/config.js
module.exports = {
  themeConfig: {
    sidebar: {
      '/foo/': [
        '',     /* /foo/ */
        'one',  /* /foo/one.html */
        'two'   /* /foo/two.html */
      ],

      '/bar/': [
        '',      /* /bar/ */
        'three', /* /bar/three.html */
        'four'   /* /bar/four.html */
      ],

      // fallback
      '/': [
        '',        /* / */
        'contact', /* /contact.html */
        'about'    /* /about.html */
      ]
    }
  }
}

    从上到下匹配，所以越短的应该放在越后面，否则将会被前面匹配

    # 更新时间

    基于git提交时间，只有提交记录的文件才会显示更新时间

    // .vuepress/config.js
module.exports = {
    themeConfig: {
        lastUpdated: 'Last Updated', // string | boolean
        }
    }

    git status 查看文件是否被管理追踪
    git add . 加入跟踪
    git commit -m"feat: about" 提交记录

    修改文字

     lastUpdated: '更新时间',

    更换时间格式，安装插件Momentm

    yarn add moment

    配置Moment

    const moment = require('moment');

module.exports = {
  plugins: [
    [
      '@vuepress/last-updated',
      {
        transformer: (timestamp, lang) => {
          moment.locale(lang)
          return moment(timestamp).fromNow()
        }
      }
    ]
  ]
}
